CSS grid vs. Flexbox layout

CSS grid is a CSS layout method which is designed for two-dimensional layout items on a webpage. CSS grid supports rows and columns at the same time which means that the CSS grid can accommodate the size for rows and columns on the webpage according to the size of the content. On the other hand, the Flexbox layout is one dimensional as it can only offer the distribution of items in either rows or columns. For my website, I used CSS grid. As I had used the flexbox layout in one of my previous projects I wanted to learn about different CSS techniques. In order to do so and expand my knowledge, I learned how the CSS grid was used and implemented it into my website. The main problem I faced during the CSS grid implementation was with the integration of the artwork and text into the column and row but after referring to www.w3school.com I found out the correct way to adjust the CSS for the grid.

HTML-Canvas vs. SVG in HTML

HTML Canvas is an element which can be created to render graphs, games and images as it is a resolution dependent bitmap while SVG is a language that allows the programmer to create two-dimensional objects which are interactive and can be interacted with by using scripts or SVG animations. Just like the flexbox layout, I had used HTML canvas before, so I taught myself SVG and implemented it into my website. After I had learned SVG, I still started with HTML canvas but I realized that it required a lot more coding than SVG as the elements had to be recreated. This made me realize that it was easier to manipulate elements and animations with SVG than canvas because SVG is meant for the interaction of two-dimensional elements so I used SVG to create the artwork for my website.

jQuery

jQuerry is a fast JavaScript library which makes traversing and manipulation of events, animations, etc. in a HTML document easier. It is an easy-to-use API which works on almost all the web-browsers. jQuerry is used to simplify Javascript code and is better written JavaScript as the programmer writes less and does more. For my website I used jQuerry for the button to show the documentation and choosing & selecting the seat on the artwork. I had previously used jQuerry but while I was doing this project I realized if I had used JavaScrip, I would have written a lot more lines than I did with jQuerry because jQuerry compiled my code into fewer lines than I had expected and it helped me a fair amount in the development of my website.

Cross-browser testing

In order to make sure that the website works on the different web browsers I conducted a test where I downloaded Internet Explorer, Edge, Opera, Safari, Google Chrome and Firefox. After installing all the web browsers, I opened my website on each browser and it worked on each browser. The reason why I did such intensive cross-browser testing is because people use different web browsers and in order to maintain the same integrity, behavior and experience for the users I conducted this test so make sure that my website works in the exact same way for each browser.
