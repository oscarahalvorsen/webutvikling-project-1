Layout

This website uses a css-grid to create the layout of the page. Grid positioning is seen as the best way to places elements in relation to each other on a 3d-grid. It removes the need to use absolute and relative positioning. This page contains four elements: “header”, “left”, “right” and “footer”. The grid is split into two columns and three rown. The header and the footer span both the columns of respectively the top and the bottom row. The middle row is however split into the right and the left element. To correctly place these four element into the grid I use the css property “grid-template-areas”.

To facilitate smaller screens this page uses “media only screen and (max-width: 1100px)”, which changes the layout of the page if the width of the screen is less than 110px. In this case the grid instead contains one column and four rows, with one row for each element. The element are thus stacked on top of each other.

SVG

The svg-image contains multiple svg-elements that are enclosed in a div-tag with id=”svg-div”. The dart board is defined in the html as one svg-element. It consists of multiple circles layered on top of each other.

The darts are however defined in a separate JavaScript file called dartsvg.js. When one clicks on svg-div a dart appears from below that moves into the point where one clicked. This is done by the function svgdiv.click(function(e) {foo}, that takes in an event e. The event e has the properties offsetX and offsetY that return the coordinates of the pointer when the event took place. Then an instance of the dart-svg is appended to svgdiv. A function called moveDart() subsequently moves the dart-svg to its correct position by using the built in jQuery method called animate().

One can also reset the svg dart board by pressing the “Reset SVG”-button, which deletes all the dart-svg with the built in jQuery method called remove().

Canvas

The canvas is almost exclusively drawn in a separate JavaScript file called dartcanvas.js. Only the canvas-element itself is defined in the html-file. The board is drawn by the function drawBoard() and the darts is drawn by the function drawDart(). To draw they use the simple built in canvas functions beginPath(), lineTo(), arc(), closePath, fill() and stroke().

The event listening is done in the same manner as the svg drawing. The animation is achieved by moving the darts pixel by pixel and drawing the whole drawing for each time the dart moves one step. This is achieved by using a for loop and a setTimout() function inside it. The coordinate of the dart is then saved in a list called dartList. From then on, each time a new dart animation is drawn, all the old darts will still be drawn since their coordinates are saved.

One can also reset the canvas dart board, by pressing the “Reset CANVAS”-button, which redraws the board and removes all elements in the dartList.

Other interactivity

The “show documentation”-button shows documentation on the page. The button text also changes to “hide documentation”. When pressed again the text changes back and the documentation disappears. This is done by using the built in jQuery functions .show() and hide(). To make the page more fluid a scroll function is added when clicked on the button, that scrolls down to the documentation so the user doesn’t have to. This is done in html and css. The button is in reality an a-tag, that moved the page to the documentation. The css attribute “scroll-behavior: smooth” in body makes the page scroll to the documentation smoothly.
